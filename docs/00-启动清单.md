# 启动清单：5 分钟开始你的个人网站

> 你不懂编程，但这不影响你做出一个专业、个性化、手机能看的个人网站。
> 这份清单是整件事的**入口**——先看它，照着做，遇到不懂的再翻另外两份文档。

---

## 一句话定位

你要做的不是一个"网页"，而是一个**产品**。
工程化 vibe coding 的核心就是：**像做产品一样做网站，而不是一句话让 AI 吐一坨代码**。

你负责想清楚"要什么"，**Codex 负责"怎么写代码"**。
这份清单 + 另外两份文档，就是你们俩之间的**合作规则**。

---

## 🗺️ 全程 9 阶段一览（先有个全景，别慌）

| 阶段 | 做什么 | 产出物 | 谁主导 |
|------|--------|--------|--------|
| **0. 初始化** | 建项目目录、装 Git、放骨架文档 | 一个干净的项目文件夹 | 你 + Codex |
| **1. 写需求 PRD** | 说清"我是谁、写给谁、有哪些页面" | `PRD.md` | **你** |
| **2. 选技术栈** | 让 Codex 推荐 3 个方案，你选 | `DECISIONS.md` 里的一条决策 | Codex 推荐 → 你定 |
| **3. 找设计参考** | 让 Codex 推荐 demo，你看完选风格 | 选定的参考站截图/链接 | Codex 推荐 → 你定 |
| **4. 设计信息架构** | 定网站地图、导航、目录结构 | `ARCHITECTURE.md` | Codex 起草 → 你确认 |
| **5. 分页面实现** | 首页 → 课程目录 → 单篇内容 → 关于我，逐页做 | 能跑的网站雏形 | Codex 写 → 你验收 |
| **6. 加轻交互** | 评论、邮件订阅、课程进度 | 可评论、可订阅的站 | Codex 写 → 你验收 |
| **7. 手机适配 + 内容** | 手机预览、填上你的真实内容 | 内容齐全、手机端好看 | 你填内容 → Codex 调样式 |
| **8. 部署上线** | 部署到 Vercel，拿到网址 | 一个公开可访问的网址 | Codex 指导 → 你点几下 |
| **9. 持续迭代** | 写新内容、备份、小优化 | 持续更新的活网站 | **你**长期维护 |

## 🚀 现在，立刻可做的 4 个动作

### 动作 ①：确认项目目录

你的网站项目就放在这个文件夹里（现在已经建好了）：

```
d:\Personal Data Huawei\Personal Data\AI\Projects\MyWebsite\
└── docs\        ← 你正在看的文档在这里
```

> ✅ 它和你电脑里其他东西是分开的，不会互相影响。以后网站的所有代码、文章，都放这个文件夹。

---

### 动作 ②：确认装了 Git（版本管理的"后悔药"）

**Git 是什么？** 简单说，它给项目拍"快照"。每写完一段，拍一张，以后随时能回到任何一张快照。这是工程化的基础——**让你永远不怕改坏**。

**怎么检查**：打开 Codex，对它说一句：

```
帮我检查这台电脑有没有装 Git。如果没有，告诉我怎么装（我是 Windows，不懂编程，请给最简单的步骤）。
```

> Git 装好之前，不要急着写代码。这是地基。

---

### 动作 ③：写 50 字自我介绍（你的 PRD 种子）

打开记事本，用大白话写下这几句（不用写得漂亮，写清楚就行）：

```
- 我是谁：（例如：一个正在系统学习 AI 的学习者，同时在带一批学员）
- 我的网站主要给谁看：（例如：我的学员 + 对 AI 感兴趣的初学者）
- 我最想放上去的 3 样东西：（例如：①我的学习笔记 ②给学员的课程目录 ③关于我）
- 我希望别人来我网站后的感觉：（例如：觉得专业、好懂、愿意常回来看）
```

> 这 50 个字，就是将来你网站需求文档（PRD）的种子。先有它，后面 Codex 才知道往哪做。
> 完整 PRD 模板见 [`01-开发路线图.md` 第 4 节](./01-开发路线图.md)。

---

### 动作 ④：用第一段话术，正式跟 Codex 开口

完成上面 3 步后，打开 Codex，**一字不差地复制**下面这段话发给它（方括号 `[...]` 里的内容换成你自己的）：

```
你好 Codex，我要做一个个人网站。背景如下：
- 我完全不懂编程，这是我的第一个网站项目。
- 网站定位：课程/教学站（有章节目录和学习路径，引导我的学员系统跟学我的 AI 学习内容）。
- 交互需求：轻交互（内容展示 + 评论 + 邮件订阅，不想上数据库和后端服务器）。
- 设计风格：我还没定，希望你先推荐一些优秀 demo 给我参考。

我已经准备好了：项目目录、Git（如果没装我会按你说的装）、一段自我介绍。

请你先不要写代码。我们的工作方式是这样的：
1. 先按这份《启动清单》和另外两份文档（开发路线图、Codex沟通手册）来，分阶段做。
2. 关键决策（技术选型、设计风格）我都希望你先推荐 2-3 个方案和 demo，我看完再选，你再执行。

我们的第一步，按《开发路线图》阶段 0 来：帮我初始化项目（建好 Git 仓库和必要的骨架文件）。
请先告诉我你打算怎么做，等我确认后再动手。
```

> 这段话术的完整版和"为什么这么说"，见 [`02-Codex沟通手册.md` 第 4 节](./02-Codex沟通手册.md)。

## 📚 三份文档怎么配合用？

| 什么时候看 | 看哪份 |
|------------|--------|
| **现在**（刚启动，想知道第一步） | `00-启动清单.md`（你正在看） |
| **想知道某个阶段具体怎么做、怎么验收** | `01-开发路线图.md` |
| **不知道怎么跟 Codex 描述需求 / 要复制话术** | `02-Codex沟通手册.md` |

> 💡 建议把这三份文档都发给 Codex 让它也读一遍——这样你们用同一套语言，配合会顺畅得多。

## ✅ 启动完成的标志

做到下面 3 件事，就算正式启动了：

- [ ] 项目目录建好（已完成）
- [ ] Git 装好、仓库初始化
- [ ] Codex 读过这三份文档，并且已经帮你做完**阶段 0（初始化）**

然后进入 [`01-开发路线图.md`](./01-开发路线图.md)，开始**阶段 1：写你的需求 PRD**。

---

**别怕慢。** 工程化的节奏就是"一步一步、每步都稳"。每完成一个阶段，你都会有一个能看见、能用、能回退的成果——这比"一口气生成一个看起来很酷但改不动"的网站，踏实一百倍。
